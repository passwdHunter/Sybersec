# SQL Injection - GBK Encoding Bypass (RootMe)

**Платформа:** RootMe  
**Категория:** Web-Client  
**Сложность:** Средняя  

##  Описание задачи
Задача основана на обходе экранирования символов в приложениях, использующих китайскую кодировку GBK.

##  Процесс решения

### 1. Понимание механики GBK
GBK (Guojia Biaozhun Kuozhan) - китайская кодировка символов, где некоторые символы занимают 2 байта. Уязвимость возникает при:
- Использовании GBK на frontend
- Использовании UTF-8/Latin1 в базе данных
- Некорректной обработке многобайтовых символов

### 2. Обход экранирования
Стандартное экранирование кавычки (`'` → `\'`) может быть обойдено с помощью специальных символов GBK:
Логин: 予' OR 1=1 --
Пароль: любой (или пусто)

### 3. Механизм bypass
Символ `予` (0xE7 0x88 0xBB) в сочетании с кавычкой создает последовательность, где слэш экранирования становится частью китайского иероглифа, оставляя кавычку активной.

### 4. Извлечение данных
После успешного обхода аутентификации, извлек пароль администратора стандартными методами SQL инъекции.


##  Ключевые уроки
- **Кодировки символов** могут создавать уязвимости безопасности
- **GBK/UTF-8 несоответствия** - известный вектор атак
- **Многобайтовые символы** могут ломать логику экранирования
- **Важность единой кодировки** во всем стеке приложения

##  Использованные инструменты
- Знание таблиц кодировок символов
- Понимание механизмов экранирования
- Тестирование специальных символов
