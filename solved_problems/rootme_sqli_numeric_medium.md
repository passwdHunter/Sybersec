# SQL Injection - Numeric Parameter (RootMe)

**Платформа:** RootMe  
**Категория:** Web-Client  
**Сложность:** Средняя  

##  Описание задачи
Форма входа с логином и паролем, требующая обхода аутентификации через SQL инъекцию.

## Процесс решения

### 1. Начальное тестирование
Изначально сосредоточился на основной форме входа:
- Тестировал базовые payload'ы: `admin' --`, `' OR '1'='1`
- Проверял различные методы обхода фильтров
- Исследовал клиентскую валидацию

### 2. Расширение области поиска
После долгих неудачных попыток, решил исследовать весь сайт более тщательно. Обнаружил "пустую" страницу с новостями, содержащую параметр:
/news.php?id=1

### 3. Обнаружение уязвимости
Параметр `id` оказался уязвимым к numeric SQL инъекции:
- `?id=1'` → ошибка SQL
- `?id=1 OR 1=1` → успешное выполнение(тащим пароль админа)

### 4. Извлечение данных
Использовал UNION-атаку для получения пароля администратора:
?id=1 UNION SELECT username,password FROM users --
