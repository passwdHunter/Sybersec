# SQL Injection - Authentication Bypass (RootMe)

**Платформа:** RootMe  
**Категория:** Web-Client  
**Сложность:** Средняя  

##  Описание задачи
Базовая задача на обход аутентификации через SQL инъекцию в форме входа.

##  Процесс решения

### 1. Начальное тестирование
Проверил форму входа базовым payload'ом:
Логин: admin' OR 1=1 --
Пароль: любой

text

### 2. Подтверждение уязвимости
После ввода payload'а получил доступ под учетной записью "user1", что подтвердило наличие SQL инъекции.

### 3. Углубленная эксплуатация
Использовал Union-based атаку для извлечения учетных данных:
Логин: admin' UNION SELECT username,password FROM users --
Пароль: a

text

### 4. Перехват и анализ
Использовал Burp Suite для перехвата запроса и анализа ответа, содержащего пароли.


##  Ключевые уроки
- **Формы аутентификации** - частый вектор атак SQL инъекции
- **Union-based атаки** эффективны для извлечения данных
- **Burp Suite** незаменим для анализа и модификации запросов
- **Базовые знания SQL** критически важны для понимания инъекций

##  Использованные инструменты
- Burp Suite Community Edition
- Ручное тестирование payload'ов
- Анализ структуры SQL запросов
