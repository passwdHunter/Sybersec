# SQL Injection - String Parameter (RootMe)

**Платформа:** RootMe  
**Категория:** Web-Client  
**Сложность:** Средняя  

##  Описание задачи
Поиск уязвимости SQL инъекции в строковом параметре с применением различных техник обхода защиты.

## Процесс решения

### 1. Исходный подход
Начал с тестирования сложных методов:
- Обход фильтрации символов
- Тестирование WAF bypass техник
- Использование кодирования payload'ов

### 2. Смена стратегии
После продолжительных неудачных попыток, решил вернуться к основам и исследовать весь функционал сайта.

### 3. Обнаружение уязвимости
Нашел страницу поиска по сайту с параметром:
/search?query=test


Параметр `query` оказался уязвимым к базовой SQL инъекции:
- `test'` → ошибка синтаксиса SQL
- `test' OR '1'='1` → успешное выполнение(тащим пароль админа)

### 4. Эксплуатация
Использовал простой Union-based payload для извлечения данных:
test' UNION SELECT null,version() --


## Ключевые уроки
- **Иногда решение проще, чем кажется** - не всегда нужны сложные bypass техники
- **Функциональность поиска часто уязвима** к SQL инъекциям
- **Важно проверять весь attack surface** приложения
- **Базовые payload'ы могут работать там, где сложные методы неэффективны**

## Использованные инструменты
- Различные техники кодирования payload'ов
- Анализ ответов сервера
- Методология постепенного усложнения тестов
